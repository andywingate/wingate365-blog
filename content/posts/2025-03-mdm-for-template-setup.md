<p></p><p></p><p>If you are managing multiple companies in a single BC environment you can use master data management (MDM) to sync your, as the name suggests, master data like Customers and Vendors.</p><p>Not every organisation needs to do that, but it's still worth remembering you can use the MDM feature to sync any table or field that's not posted data.</p><p>In the example below I show a simple setup to use MDM to sync key fields from the General Ledger Setup table - to manage the <b>Allowed Posting Dates</b> centrally.</p><p>As always lets check the docs&nbsp;<a href="https://learn.microsoft.com/en-us/dynamics365/business-central/admin-set-up-data-sync">Set up companies to synchronize master data - Business Central | Microsoft Learn</a>&nbsp;here are the key steps from that page:&nbsp;</p><h3 data-end="329" data-start="265"><strong data-end="329" data-start="272">Quick Recap: Setting Up Data Sync in Business Central</strong></h3><ul data-end="1351" data-start="515"><li data-end="735" data-start="515"><p data-end="735" data-start="517"><strong data-end="547" data-start="517">Pull Model:&nbsp;</strong><span data-end="547" data-start="517">&nbsp;The BC master data management tool allows you to pull data from a source company. Bi-directional sync is not possible out-of-the-box.</span></p></li><li data-end="735" data-start="515"><p data-end="735" data-start="517"><strong data-end="547" data-start="517">Integration Table Mappings</strong>: These define how data flows between Business Central and Dataverse. Predefined mappings exist for common entities (like customers and contacts), and you can customise or create new ones.</p>
</li>
<li data-end="915" data-start="737">
<p data-end="915" data-start="739"><strong data-end="759" data-start="739">Coupling Records</strong>: Before synchronising, you need to <strong data-end="805" data-start="795">couple</strong> (link) records so that the system knows they represent the same thing.</p>
</li>
<li data-end="1033" data-start="917">
<p data-end="1033" data-start="919"><strong data-end="933" data-start="919">Job Queues</strong>: Use job queues to <strong data-end="981" data-start="953">automate synchronisation</strong>. You can define how often data is pushed or pulled.</p>
</li>
<li data-end="1201" data-start="1035">
<p data-end="1201" data-start="1037"><strong data-end="1066" data-start="1037">Manual vs. Automatic Sync</strong>: Some syncs happen automatically (via job queues), while others can be triggered manually, especially during setup or troubleshooting.</p>
</li>
<li data-end="1351" data-start="1203">
<p data-end="1351" data-start="1205"><strong data-end="1229" data-start="1205">Managing Sync Errors</strong>: Business Central logs sync issues in the <strong data-end="1307" data-start="1272">Integration Synchronisation Log</strong>, which helps diagnose and resolve problems.</p></li></ul><h3 style="text-align: left;">Setup Steps</h3><div>In the steps below <b>TARGET </b>is the company that will pull the data, <b>SOURCE </b>is the company where that data will be pulled from.&nbsp;</div><div><ul style="text-align: left;"><li>Visit the Master Data Management page in the <b>TARGET </b>company [1]&nbsp;</li></ul></div><div><div class="separator" style="clear: both;
