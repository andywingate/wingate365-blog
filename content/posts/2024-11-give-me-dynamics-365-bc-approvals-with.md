<p></p><p>&nbsp;This is something I'm asked on most projects when we get to the topic of approvals.</p><p></p><p></p><ul style="text-align: left;"><li>Client: "WOW these BC approval workflows look good, and I expect the approval will be automatically delegated if the user has their outlook out-of-office on, right?"</li><li>Me: "About that..."</li></ul><div>Contina does a good effort on managing out-of-office&nbsp;<a href="https://docs.continia.com/en-us/continia-document-capture/setting-up-document-capture/setting-up-document-approval/approval-sharing">Setting up Approval Sharing</a>, but it's a config that's inside BC, and it doesn't cover all document types.&nbsp;</div><div><br /></div><div>Most users simply <b>expect </b>out-of-office to be something you setup once in outlook and it's shared to all the other MS apps.</div><div><br /></div><div>I've confessed in the past that I rather use BC approvals over Power Automate approvals. Possibly that's driven by me simply being able to setup and test anything from simple to big hairy approvals in BC a lot faster than in PA.</div><div><br /></div><div>Yes, there is an automatic delegation in BC workflows. It works on a count down, it will auto delegate after x many days. But it is very rarely used, if ever. Why? Consider the real word situation. Someone asked you to approve something, but you can't right away. Could be a number of reasons why; you need to look something up, check with someone who is away etc etc. Maybe you just need to sit on it for a bit. The last thing you want is the approval going to someone else in these situations!</div><div><br /></div><div>To solve the problem above I wanted to take a blended approach; use BC for the approvals and PA for its low-code access to Exchange Online / Outlook for the out of office status of a user.</div><h2 style="text-align: left;">Solution</h2><div><ul style="text-align: left;"><li>Create Approval workflow of any kind in BC <b>[Config]</b></li><ul><li><a href="https://learn.microsoft.com/en-gb/dynamics365/business-central/across-set-up-workflows">Set up approval workflows - Business Central | Microsoft Learn</a></li></ul><li>Create API query for approval entries &amp; users emails&nbsp;<b>[AL code]</b></li><ul><li><a href="https://yzhums.com/57475/">Dynamics 365 Business Central: API query type (Develop a custom API using Query) | Dynamics 365 Lab</a></li></ul><li>Create API page for the 'delegate' action as an OData bound action <b>[AL code]&nbsp;</b></li><ul><li><a href="https://yzhums.com/39196/">Dynamics 365 Business Central: How to use Standard/Custom API Bound Actions | Dynamics 365 Lab</a></li></ul><li>Create a PA flow that checks the Outlook Out-of-Office status of the approver of open approval entries and delegates these <b>[Low-code]</b></li><ul><li><a href="https://tldr-dynamics.com/blog/check-ooo-power-automate">tldr-dynamics | Check user's out of office status using Power Automate</a></li><li><b>And this thank you FlowJoe!&nbsp;</b><a href="https://www.youtube.com/watch?app=desktop&amp;v=wtWn8XpmGys">Easily Avoid Apply to Each Loops | Power Automate</a></li></ul></ul></div><div>The key part here was to have PA fetch only the open approval entries and then loop though and only delegate those where the approval user was sending automatic replies / had set out-of-office.</div><div><br /></div><div>This solution was put together with help from my good buddy Lee Noble who sorted the code for the OData bound action.
